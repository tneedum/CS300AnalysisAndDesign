# CS300AnalysisAndDesign

In this course, the main problem I was solving was how to efficiently store, organize, and retrieve course data for a computer science advising system. The goal was to take raw data from a file, structure it using appropriate data structures, and allow users to search, sort, and display course information in a meaningful way. This included handling prerequisites, organizing course lists, and making sure the program could scale as more data was added.
To approach this problem, I focused on understanding how different data structures work and why they matter. I compared options like vectors, hash tables, and binary search trees to determine which would be most efficient for storing and retrieving course data. For example, I learned that binary search trees allow for faster searching and sorting compared to basic lists, especially when dealing with larger datasets. This helped me make more informed decisions about performance, including run-time and memory usage.
One of the biggest challenges I faced was breaking down the logic into clear steps, especially when writing pseudocode for file parsing, prerequisite validation, and course sorting. At first, I understood the overall idea, but I had to go back and make each step more explicit to fully meet the project requirements. I overcame this by carefully reviewing the rubric, revising my pseudocode, and making sure each part of the process was clearly defined instead of implied.
Working on these projects has improved the way I approach designing software. I now think more about how data is structured before I start coding, instead of jumping straight into writing code. I also consider how the program will perform as the amount of data grows, which is something I did not focus on as much before this course.
This project also helped me improve how I write maintainable and readable code. I learned the importance of organizing code into logical sections, using clear naming conventions, and writing code that can be easily updated or expanded later. For example, structuring the program with separate functions for loading data, printing courses, and searching made the code easier to understand and modify. Overall, this experience has made me more confident in writing programs that are not only functional but also clean and adaptable.
